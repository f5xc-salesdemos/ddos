## XC (UI)

[L3/L4 ルーテッド DDoS ミティゲーション](https://docs.cloud.f5.com/docs-v2/ddos-and-transit-services/how-tos/network-firewall/l3l4-ddos-mitigation)ガイドに基づいて、Web インターフェースを使用して**クラウド**側を設定します。

### DDoS ワークスペースの有効化

トンネルと BGP を設定する前に：

1. クラウドチーム（`sales@example.com`
   またはアカウントチーム経由）に連絡して以下を依頼します：
    - テナントの**ルーテッド DDoS ミティゲーション**ワークスペースを有効化。
    - 以下の情報を提供：
        - **パブリック IP ネットブロック**（IPv4 は /24 以上、IPv6 は /48 以上）および所有権の証明（または LOA）。
        - **ASN**（ARIN または同等のレジストリから発行されたもの）。
        - **IRR** 登録（RIPE、ARIN、APNIC など）および RPKI リポジトリ内の **ROA**。
        - 希望する**クリーントラフィック返送方法**（GRE トンネル、レイヤー 2、Equinix 経由のプライベートピアリングなど）。
        - 保護対象のデータセンターの場所とルーター。
        - ルートアナウンスメントに必要な **AS-Path プリペンド**。
        - **BGP コミュニティ**、ルートプリファレンス、または **AS-SET**

1. SOC が以下をプロビジョニングします：
    - トンネル
    - ASN
    - 保護対象プレフィックス
    - ルートアドバタイズメントオプション
    - 必要に応じた高速 ACL / ファイアウォールポリシー

### トンネル

1. [Cloud Console](https://docs.cloud.f5.com/docs-v2/ddos-and-transit-services/how-tos/network-firewall/l3l4-ddos-mitigation) にログインし、サービスセレクターから**ルーテッド DDoS** を選択します。
1. **Manage > Tunnels > Add Tunnel** に移動し、以下を設定します：
    - **Location Name** と **Availability Zone**（デフォルトは Zone 1）。
    - **Bandwidth Max in MB**。
    - **Tunnel Type**：
        - **GRE Over IPv4**: IPv4 外部トンネル用。
        - **GRE Over IPv6**: IPv6 外部トンネル用。
        - **IP Over IP**: IPv4-in-IPv4 カプセル化用。
        - **IPv6 Over IPv6**: IPv6-in-IPv6 カプセル化用。
    - **Customer Endpoint IP**: BIG-IP の外部アドレス（外部セルフ IP、トンネルがパブリックインターネットを経由する場合はパブリックにルーティング可能である必要があります）。
    - オプション：IPv4/IPv6 インターコネクト、フラグメンテーション、キープアライブ（すべてデフォルトで無効）。
1. **Tunnel BGP Information** で：
    - **ASN オブジェクト**（ご自身の ASN）を選択します。
    - **Customer Peer Secret Override** を設定：デフォルトシークレットを使用（デフォルト）、BGP パスワードオーバーライド（ブラインドフォールドまたはクリアテキスト）、またはシークレットなし。
    - デフォルトと異なる場合は **Holddown Timer** の値を秒単位で設定します。

SOC がこれらのトンネルオブジェクトを事前に作成する場合があります。その場合は、BIG-IP 上で**エンドポイント IP** と BGP 設定を一致させるだけです。

### ASN とルート

- **ASN**: -> **Manage > ASNs > Add ASN**。
    - ASN を入力し、BGP が有効になっていることを確認します。

- **プレフィックス**: -> **Manage > Prefixes > Add Prefix**。
    - 各 IP プレフィックスを入力し、ASN に関連付けます。

- **ルートアドバタイズメント** -> **Manage > Route Advertisement > Add Route Advertisement**。
    - プレフィックスを入力し、**Active** または **Not Advertised** を選択し、オプションで有効期限を設定します。

これらのオブジェクトは、サービスがアクティブな場合にグローバルネットワーク経由でアナウンスされるプレフィックスを制御します。

### ネットワーク制限

**ファイアウォールルール**、**拒否リストルール**、および**インターネット VIP 用高速 ACL** を使用すると、以下が可能です：

- 特定のトラフィックをブロックまたは許可する。
- 悪意のあるソースをレート制限する。
- 純粋なボリューメトリックスクラビング以上の追加 DDoS 保護を適用する。
