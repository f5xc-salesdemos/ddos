## Verifica

### BIG-IP

#### Tunnel

Eseguire su ciascuna unità per i propri tunnel:

```shell title="tmsh"
show net tunnels tunnel xc-ddos-c1t1-v4
show net tunnels tunnel xc-ddos-c1t1-v6
show net tunnels tunnel xc-ddos-c1t2-v4
show net tunnels tunnel xc-ddos-c1t2-v6
show net tunnels tunnel xc-ddos-c2t1-v4
show net tunnels tunnel xc-ddos-c2t1-v6
show net tunnels tunnel xc-ddos-c2t2-v4
show net tunnels tunnel xc-ddos-c2t2-v6
list net tunnels tunnel xc-ddos-c1t1-v4 all-properties
```

#### Self IP

```shell title="tmsh"
list net self xc-ddos-*
```

#### Verifica del percorso

Da ciascuna unità:

```shell title="tmsh"
# BIG-IP-A tunnels
ping xXC_C1_T1_INNER_V4x source xBIGIP_C1_T1_INNER_V4x
ping xXC_C2_T1_INNER_V4x source xBIGIP_C2_T1_INNER_V4x

# BIG-IP-B tunnels
ping xXC_C1_T2_INNER_V4x source xBIGIP_C1_T2_INNER_V4x
ping xXC_C2_T2_INNER_V4x source xBIGIP_C2_T2_INNER_V4x
```

#### BGP

In imish su ciascuna unità:

```shell title="imish"
show ip bgp summary
show ipv6 bgp summary
show ip bgp
show ipv6 bgp

# BIG-IP-A neighbors
show ip bgp neighbors xXC_C1_T1_INNER_V4x
show ip bgp neighbors xXC_C2_T1_INNER_V4x
show ip bgp neighbors xXC_C1_T1_INNER_V4x advertised-routes
show ip bgp neighbors xXC_C1_T1_INNER_V4x received-routes

# BIG-IP-B neighbors
show ip bgp neighbors xXC_C1_T2_INNER_V4x
show ip bgp neighbors xXC_C2_T2_INNER_V4x
show ip bgp neighbors xXC_C1_T2_INNER_V4x advertised-routes
show ip bgp neighbors xXC_C1_T2_INNER_V4x received-routes

show ip route
show ipv6 route
```

:::caution[Checklist per la risoluzione dei problemi]
Se il BGP non si stabilisce:

1. Verificare che il tunnel GRE sia attivo: `show net tunnels tunnel` dovrebbe
   mostrare lo stato `up`.
1. Confermare che `allow-service` includa `tcp:179` sul self IP interno.
1. Eseguire il ping dell'IP interno remoto attraverso il tunnel.
1. Verificare che la password BGP corrisponda su entrambi i lati (causa comune
   degli errori `NOTIFICATION`).
1. Verificare che le prefix-list e le route-map siano applicate correttamente.
1. Controllare `show ip bgp neighbors <IP>` per l'ultimo errore/motivo
   di reset.
1. Su ciascuna unità HA, verificare che i tunnel dell'unità stessa siano attivi e che
   i suoi neighbor BGP siano configurati (non quelli dell'altra unità).
:::
