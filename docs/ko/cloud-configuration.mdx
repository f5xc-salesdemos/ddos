## XC (UI)

[L3/L4 라우팅 기반 DDoS 완화](https://docs.cloud.f5.com/docs-v2/ddos-and-transit-services/how-tos/network-firewall/l3l4-ddos-mitigation) 가이드를 기반으로 웹 인터페이스를 사용하여 **클라우드** 측을 구성합니다.

### DDoS 워크스페이스 활성화

터널과 BGP를 구성하기 전에:

1. Cloud 팀(`sales@example.com` 또는 담당 계정 팀)에 연락하여:
    - 테넌트에 대한 **라우팅 기반 DDoS 완화** 워크스페이스를 활성화합니다.
    - 다음 정보를 제공합니다:
        - **공인 IP 넷블록** (IPv4의 경우 /24 이상, IPv6의 경우 /48 이상) 및 소유권 증명(또는 LOA).
        - **ASN** (ARIN 또는 동등한 레지스트리에서 발급받은 것이어야 함).
        - **IRR** 등록(RIPE, ARIN, APNIC 등) 및 RPKI 리포지토리의 **ROA**.
        - 원하는 **클린 트래픽 반환 방법** (GRE 터널, 레이어 2, Equinix를 통한 프라이빗 피어링 등).
        - 보호할 데이터 센터 위치 및 라우터.
        - 라우트 공고에 필요한 **AS-Path 프리펜드**.
        - **BGP 커뮤니티**, 라우트 선호도 또는 **AS-SET**

1. SOC에서 다음을 프로비저닝합니다:
    - 터널
    - ASN
    - 보호 대상 프리픽스
    - 라우트 광고 옵션
    - 필요에 따라 Fast ACL / 방화벽 정책

### 터널

1. [Cloud Console](https://docs.cloud.f5.com/docs-v2/ddos-and-transit-services/how-tos/network-firewall/l3l4-ddos-mitigation)에 로그인하고 서비스 선택기에서 **Routed DDoS**를 선택합니다.
1. **Manage > Tunnels > Add Tunnel**로 이동하여 다음을 구성합니다:
    - **Location Name** 및 **Availability Zone** (기본값은 Zone 1).
    - **Bandwidth Max in MB**.
    - **Tunnel Type**:
        - **GRE Over IPv4**: IPv4 외부 터널용.
        - **GRE Over IPv6**: IPv6 외부 터널용.
        - **IP Over IP**: IPv4-in-IPv4 캡슐화용.
        - **IPv6 Over IPv6**: IPv6-in-IPv6 캡슐화용.
    - **Customer Endpoint IP**: BIG-IP의 외부 주소(외부 self IP, 터널이 공용 인터넷을 통과할 때 공개적으로 라우팅 가능해야 함).
    - 선택 사항: IPv4/IPv6 상호 연결, 프래그먼테이션, 킵얼라이브(모두 기본적으로 비활성화됨).
1. **Tunnel BGP Information** 아래에서:
    - **ASN object**(사용자의 ASN)를 선택합니다.
    - **Customer Peer Secret Override**를 설정합니다: Use Default Secret(기본값), BGP Password Override(블라인드 또는 일반 텍스트) 또는 No Secret.
    - 기본값과 다른 경우 **Holddown Timer** 값을 초 단위로 설정합니다.

SOC에서 이러한 터널 객체를 미리 생성할 수 있으며, BIG-IP에서 **엔드포인트 IP** 및 BGP 설정을 일치시키기만 하면 됩니다.

### ASN 및 라우트

- **ASN**: -> **Manage > ASNs > Add ASN**.
    - ASN을 입력하고 BGP가 활성화되어 있는지 확인합니다.

- **프리픽스**: -> **Manage > Prefixes > Add Prefix**.
    - 각 IP 프리픽스를 입력하고 ASN과 연결합니다.

- **라우트 광고** -> **Manage > Route Advertisement > Add Route Advertisement**.
    - 프리픽스를 입력하고, **Active** 또는 **Not Advertised**를 선택하며, 선택적으로 만료 기간을 설정합니다.

이러한 객체는 서비스가 활성화되었을 때 글로벌 네트워크를 통해 공고되는 프리픽스를 제어합니다.

### 네트워크 제한

**Firewall Rules**, **Deny List Rules** 및 **Fast ACLs for Internet VIPs**를 통해 다음을 수행할 수 있습니다:

- 특정 트래픽을 차단하거나 허용합니다.
- 악의적인 소스에 대해 속도를 제한합니다.
- 순수 볼류메트릭 스크러빙 이상의 추가적인 DDoS 보호를 적용합니다.
